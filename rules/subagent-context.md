# Subagent Context — правила передачи контекста

## Принцип

```
Нужен субагент?
       │
       ├─ Задача автономная (найти, прочитать, суммаризировать)?
       │   └─ НЕ передавать контекст → субагент работает сам
       │
       ├─ Задача требует знания текущего состояния?
       │   └─ Передать МИНИМАЛЬНЫЙ контекст (только нужное)
       │
       └─ После выполнения:
           ├─ summary в main context (кратко)
           └─ verbose output остаётся в субагенте
```

---

## Когда НЕ передавать контекст

```
· Поиск файлов / grep по кодовой базе
· Чтение и суммаризация документации
· Code review кода который субагент читает сам
· Анализ структуры директорий
· Fetch внешних URL / документации
```

Пример (правильно):
```python
Agent(
    subagent_type='codebase-explorer',
    prompt='Найди все файлы с функцией send_message в src/'
    # context НЕ нужен — агент читает файлы сам
)
```

---

## Когда передавать минимальный контекст

```
· Субагент должен знать текущий файл который редактируется
· Субагент должен знать конкретные константы / состояние
· Задача продолжает предыдущую без возможности восстановить из кода
```

Пример (правильно):
```python
Agent(
    subagent_type='codebase-explorer',
    prompt=f'''
    Текущий контекст: работаем с файлом {current_file}, правим функцию {func_name}.
    Задача: найди все вызовы этой функции и их параметры.
    '''
    # только минимально необходимое
)
```

---

## После выполнения субагента

```
Субагент вернул результат
       │
       ├─ Нужен в следующих задачах? ДА → compact summary в main context
       │                              НЕТ → игнорировать подробности
       │
       └─ Verbose output → в субагенте, НЕ копировать в main
```

### Как суммаризировать правильно

```
Плохо:  вернуть весь вывод субагента в main context
Хорошо: "Субагент нашёл 3 файла: auth.py, db.py, api.py"
```

## Изоляция через worktree

Для рискованных экспериментов:
```python
Agent(
    subagent_type='general-purpose',
    isolation='worktree',  # изолированная копия репо
    prompt='Попробуй рефакторинг, если плохо — не мержим'
)
# Worktree автоматически очищается если изменений нет
```

## Параллельные субагенты

```python
# Независимые задачи — запускать В ОДНОМ сообщении
Agent(..., run_in_background=True)  # агент 1
Agent(..., run_in_background=True)  # агент 2

# Зависимые задачи — последовательно, ждать результат первого
result1 = Agent(...)  # дождаться
Agent(..., prompt=f'Используй результат: {result1}')
```

**Правило:** максимум 3 субагента параллельно. Main context должен оставаться чистым.

---

## Очистка после задачи

```
Задача субагента завершена?
       │
       ├─ Summary получен → удалить из context подробности субагента
       ├─ Verbose логи → не хранить в main context
       └─ Если задача закончена → /compact или /clear перед следующей
```
