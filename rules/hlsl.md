## Универсальные принципы (применимы к любому шейдеру)

### Структура правки
```
1. Прочитать CLAUDE.md проекта → найти что можно/нельзя трогать
2. Правка только параметров, не топологии (если не указано явно)
3. Деплой через install.ps1 -NoPrompt
4. Проверить в MPC-BE руками
```

### Что НЕЛЬЗЯ без явного запроса
- Топология детектора (пороги срабатывания, логика детекции)
- Порядок применения эффектов
- Балансные значения найденные итерацией (отмечены LOCKED в комментариях)

### Вайбкодинг → hlsl
При промпте "сделай теплее/ярче/тише/плавнее":
- Найти нужные константы через `vibe_coding.md` маппинг проекта
- Изменить ТОЛЬКО их
- НЕ трогать детектор, NOT трогать ps_3_0 instruction count без необходимости

### Откат
```powershell
# К prev (последнему рабочему):
Copy-Item "shader.prev.hlsl" -Destination "shader.hlsl" -Force
# К backup (эталону):
Copy-Item "shader.backup.hlsl" -Destination "shader.hlsl" -Force
```
