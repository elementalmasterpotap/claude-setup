Проанализируй текущую сессию и найди паттерны которые стоит зафиксировать как правила.

## Что искать:

```
├── Нетривиальные решения которых нет в lessons_universal.md
├── Повторяющиеся паттерны взаимодействия с Потапом
├── Ловушки/антипаттерны найденные в этой сессии
├── Предпочтения Потапа которые ещё не задокументированы
├── Правила которые применялись неявно — стоит сделать явными
└── Возможности для новых хуков или скиллов которых пока нет
```

## Формат ответа — таблица предложений:

| # | Паттерн | Уровень | Где хранить | Hook? | Skill? |
|---|---------|---------|-------------|-------|--------|
| 1 | ... | глобальный / проектный | rules/*.md | да/нет | да/нет |

## После таблицы:

Для каждого пункта — одна строка с обоснованием почему стоит добавить.

Потап выбирает что применять → `/learn [описание]` для каждого выбранного.

## Дополнительно — аудит дыр в системе:

Проверь есть ли правила в rules/ без хука И без скилла (нет ни того ни другого).
Покажи их отдельным списком — возможно стоит добавить.
