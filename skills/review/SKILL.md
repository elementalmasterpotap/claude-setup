# /review — Система самообучения

**Фокус:** $ARGUMENTS (если пусто — анализируй все категории)

## Алгоритм

### 1. СБОР ДАННЫХ

Прочитай (параллельно):
- `~/.claude/tasks/SESSION_PATTERNS.md` — пассивный лог паттернов из session-pattern-logger.py
- `~/.claude/SESSION.md` — лог файлов из checkpoint.py

Проанализируй текущий диалог в памяти — ищи:
- Что пользователь исправлял после твоего ответа ("нет", "не так", "неправильно")
- Что Claude делал несколько раз одинаково (повторяющийся подход)
- Какие команды/паттерны запускались 2+ раз
- Что пользователь говорил о своих предпочтениях
- Где не хватало инструмента (приходилось делать руками или через костыль)

### 2. ФОРМИРОВАНИЕ ПРЕДЛОЖЕНИЙ

Для каждого найденного паттерна сформируй запись:

```
тип        — [ХУК] / [ПРАВИЛО] / [СКИЛЛ] / [ПАМЯТЬ] / [ПЛАГИН]
что        — конкретное предложение (1 строка)
пример     — что именно произошло в сессии (цитата или описание)
рекомендация — ГЛОБАЛЬНО или ЛОКАЛЬНО + почему
```

**Правила категоризации:**

| Если... | То тип | Куда |
|---------|--------|------|
| Claude делал X неправильно 2+ раз | ХУК (Stop) | глобально |
| Обсудили новое соглашение/правило | ПРАВИЛО | зависит от специфики |
| Команда/паттерн повторялась 3+ раз | СКИЛЛ | глобально |
| Потап выразил предпочтение | ПАМЯТЬ | MEMORY.md |
| Приходилось делать вручную то что мог инструмент | ПЛАГИН | глобально |
| Новая конвенция специфична для проекта | ПРАВИЛО | локально |

**ГЛОБАЛЬНО** = `~/.claude/` — работает во всех проектах
**ЛОКАЛЬНО** = `.claude/` в текущем проекте

### 3. ВЫВОД ТАБЛИЦЫ

Покажи таблицу предложений (все найденные, приоритет сверху):

```
╔══════════════════════════════════════════════════════════════════════╗
║  /review — результаты анализа сессии                                ║
╠═══╤══════════╤══════════════════════════════════════╤═══════════════╣
║ # │ Тип      │ Предложение                          │ Куда          ║
╠═══╪══════════╪══════════════════════════════════════╪═══════════════╣
║ 1 │ ХУК      │ Блокировать X если Y                │ глобально     ║
║ 2 │ ПРАВИЛО  │ Добавить в communication.md: Z       │ глобально     ║
║ 3 │ СКИЛЛ    │ Создать /deploy-report               │ глобально     ║
║ 4 │ ПАМЯТЬ   │ Потап предпочитает A в ситуации B    │ MEMORY.md     ║
╚═══╧══════════╧══════════════════════════════════════╧═══════════════╝
```

Под таблицей — краткий комментарий по каждому пункту (пример из сессии).

Если предложений 0 — сказать честно: "Чистая сессия, паттернов для фиксации нет."

### 4. ИНТЕРАКТИВНЫЙ ВЫБОР

Используй AskUserQuestion с multiSelect: true.

Если предложений ≤ 4:
- Один вопрос: все предложения как варианты + "Пропустить всё"

Если предложений > 4:
- Первый раунд: топ-4 по приоритету (хуки > правила > скиллы > память)
- После применения спросить: "Показать ещё N предложений?"

**Формулировка вопроса:** "Что применяем из этой сессии?"

Для каждого выбранного предложения с неочевидным местом (ГЛОБАЛЬНО vs ЛОКАЛЬНО):
- Задать отдельный вопрос или включить в формулировку варианта

### 5. ПРИМЕНЕНИЕ

Выполняй последовательно для каждого выбранного:

**[ХУК]:**
- Если паттерн простой (regex в тексте ответа) → создай `~/.claude/scripts/<name>-check.py`
  по образцу `anti-ration.py` (stdin JSON → проверка → block/warn)
- Добавь в settings.json в массив Stop[]
- Или: вызови `/hookify <name>` если достаточно файла-правила

**[ПРАВИЛО]:**
- Определи по карте правил из CLAUDE.md куда писать
- Edit нужного файла — добавить сжато, без воды
- Если новый файл → создать `rules/<name>.md`

**[СКИЛЛ]:**
- Создать `~/.claude/skills/<name>/SKILL.md` (или `.claude/skills/<name>/SKILL.md` если локально)
- Тип: `disable-model-invocation: true` (action) или `user-invocable: false` (knowledge)

**[ПАМЯТЬ]:**
- Edit `~/.claude/memory/MEMORY.md`
- Сначала проверить нет ли уже похожей записи (избегать дублей)

**[ПЛАГИН]:**
- Объяснить как добавить в settings.json mcpServers
- Указать конкретное имя пакета если известно
- НЕ добавлять автоматически — только рекомендовать

### 6. ФИНАЛ

После применения всего выбранного:

1. Если изменились `CLAUDE.md`, `settings.json`, `rules/*.md` → запустить:
   ```
   python3 ~/.claude/update_telegraph.py
   ```

2. Показать итог:
   ```
   ╔══════════════════════════════════════╗
   ║  Применено: N из M предложений      ║
   ╠══════════════════════════════════════╣
   ║  ✓ [ХУК] название                  ║
   ║  ✓ [ПРАВИЛО] название              ║
   ║  ✗ [СКИЛЛ] пропущено              ║
   ╚══════════════════════════════════════╝
   ```

3. Очистить SESSION_PATTERNS.md (применённые паттерны уже зафиксированы):
   ```
   # Оставить заголовок, удалить строки которые были применены
   ```

## Приоритет предложений

```
1. ХУК        — если Claude делал что-то неправильно → блокировать сразу
2. ПРАВИЛО     — соглашения которые важны для будущей работы
3. СКИЛЛ      — экономия времени на повторяющихся паттернах
4. ПАМЯТЬ     — факты о Потапе которые нужны в следующих сессиях
5. ПЛАГИН     — рекомендация, не автоматическое действие
```

## Что НЕ предлагать

- Не дублировать то что уже есть в rules/*.md / CLAUDE.md / MEMORY.md
- Не предлагать хук если паттерн был один раз (нужно 2+)
- Не предлагать скилл для одноразовой команды
- Не предлагать плагин без чёткого use-case из сессии
