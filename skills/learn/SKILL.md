Зафикси паттерн как правило: $ARGUMENTS

## Шаг 1 — определить уровень

```
Глобальное (→ ~/.claude/):
  · применимо в любом проекте
  · про технологию: PS / C# / TG / GH / Telegraph / стиль
  · поведенческий паттерн Потапа

Проектное (→ .claude/ текущего репо):
  · специфика архитектуры этого репо
  · конкретные файлы / функции / домен
  · правило "не трогать X в этом проекте"
```

Покажи пользователю свою оценку и **спроси подтверждение** перед применением:
«Это [глобальное / проектное правило] — добавлю в [файл]. Всё верно?»

## Шаг 2 — после подтверждения

По карте правил CLAUDE.md определи нужный файл, затем:

```
├── Написать правило (сжато: ID · ловушка → решение)
├── Stop hook — возможно автоматизировать?
│   ДА → создать scripts/<name>-check.py + зарегистрировать в settings.json
│   НЕТ → написать # hook: невозможно — <причина> в файле правила
└── Skill — есть смысл как команда или knowledge?
    ДА → создать skills/<name>/SKILL.md
    НЕТ → написать # skill: невозможно — <причина> в файле правила
```

## Шаг 3 — синхронизировать

```bash
python3 ~/.claude/update_telegraph.py
```

## Формат правила (коротко):

```
### <ID> — <название>
**Ловушка:** <что происходит>
**Решение:** <как правильно>
```
